# Техническое Задание: Механизм создания, регистрации плагинов TinyMCE и иконок

## 1. Введение

Данный документ объединяет стандарты и лучшие практики по разработке плагинов для TinyMCE в проекте Aurora Admin. Он охватывает весь цикл: от базовой регистрации скрипта до реализации сложной логики вставки контента и управления кастомными иконками.

---

## 2. Архитектура интеграции

### 2.1. Точка входа в Angular

В проекте используется компонент-обертка `tinymce-control.component.ts`. Все манипуляции с подключением плагинов производятся **только в нём**.

**Файл:** `src\assets\controls\tinymce-control\tinymce-control.component.ts`

**Процесс подключения:**

1.  **Plugins (строка):** Добавить имя плагина в список активных.
2.  **External Plugins (объект):** Указать путь к JS-файлу плагина.
3.  **Toolbar (массив):** Разместить кнопку плагина в интерфейсе.

```typescript
// Пример конфигурации
export class AvTinymceControlComponent {
  @Input() plugins: string = "... av-novo-plugin ...";

  @Input() external_plugins: any = {
    "av-novo-plugin": "/assets/tinymce/plugins/av-novo-plugin/plugin.js",
  };

  @Input() toolbar: string[] = ["... | av-btn-novo-action | ..."];
}
```

### 2.2. Структура `plugin.js`

Каждый плагин представляет собой самодостаточный JS-файл (Module Pattern).

```javascript
(function () {
  "use strict";

  tinymce.PluginManager.add("av-novo-plugin", function (editor) {
    // 1. Регистрация иконок
    // 2. Регистрация кнопок/меню
    // 3. Логика работы

    return {
      getMetadata: function () {
        return { name: "Aurora Plugin", url: "..." };
      },
    };
  });
})();
```

## 3. Работа с Иконками

### 3.1. Требования к SVG

Для обеспечения корректного отображения иконки должны соответствовать стандарту (см. `@docs\@doc-icons\Техническое_Задание_Генерация_SVG_Иконок.md`):

- **ViewBox:** `0 0 24 24`
- **Стиль:** Отсутствие атрибутов `transform`, использование `fill="currentColor"`.
- **Размер:** 24x24 px.

### 3.2. Регистрация в плагине

Иконки регистрируются глобально для экземпляра редактора перед созданием кнопок.

```javascript
editor.ui.registry.addIcon("av-icon-name", '<svg width="24" height="24" viewBox="0 0 24 24" ...><path d="..." fill="currentColor"/></svg>');
```

**Правило именования:** Всегда используйте префикс `av-` для избежания конфликтов с системными иконками (например, `av-align-left`).

## 4. Создание UI Элементов (Кнопки)

После регистрации иконки создается кнопка, которая ссылается на эту иконку.

```javascript
editor.ui.registry.addButton("av-btn-name", {
  icon: "av-icon-name", // Имя зарегистрированной ранее иконки
  tooltip: "Подсказка при наведении",
  onAction: function () {
    // Логика нажатия
    console.log("Button clicked");
  },
});
```

## 5. Реализация сложной логики (Сценарий "Вставка изображения")

Для сложных плагинов (например, редактор изображений) используется паттерн взаимодействия:
**Кнопка -> Модальное окно -> Обработка -> Вставка HTML**.

### 5.1. Поток данных (Data Flow)

1.  **Открытие:** Пользователь жмет кнопку в Toolbar -> Запускается `plugin.js` -> Открывается кастомное модальное окно.
2.  **Обработка:** В модальном окне происходит редактирование (Canvas API).
3.  **Экспорт:**
    - Генерация финального Canvas (с учетом слоев, рамок, теней).
    - Преобразование в Blob/DataURL.
    - (Опционально) Загрузка на сервер через `fetch`.
4.  **Генерация HTML:** Создание строки `<figure><img ...><figcaption>...</figure>`.
5.  **Вставка:** Вызов `editor.insertContent(html)`.

### 5.2. Пример реализации вставки

Функция построения HTML-фрагмента:

```javascript
// Генерация безопасного HTML для вставки
function buildHtml(data) {
  const { imageUrl, alt, width, alignment } = data;

  let styles = [];
  if (alignment === "left") styles.push("float: left; margin-right: 15px;");
  if (alignment === "right") styles.push("float: right; margin-left: 15px;");
  if (width) styles.push(`width: ${width};`);

  return `
    <figure class="aurora-image" style="${styles.join(" ")}">
      <img src="${imageUrl}" alt="${alt}" />
      <figcaption>Подпись</figcaption>
    </figure>
  `;
}

// Использование в onAction
editor.ui.registry.addButton("insert-image", {
  onAction: function () {
    // ... получение данных из модалки ...
    const html = buildHtml({ imageUrl: "...", alt: "...", alignment: "center" });
    editor.insertContent(html);
  },
});
```
