# Debug Panel - Система отладки модальных окон

## 🎯 Назначение

Debug панель позволяет отслеживать в реальном времени координаты и размеры модального окна, а также **записывать историю всех изменений** для анализа ошибок.

## 🔍 Как включить

1. Откройте модальное окно VS Modal Compromise
2. Нажмите кнопку **🔍** в заголовке окна
3. Debug панель появится в правом верхнем углу

## 📊 Что отображается

### Текущее состояние:
- **Position**: `top` и `left` - координаты верхнего левого угла
- **Size**: `width × height` - размеры окна
- **Bounds**: `right` и `bottom` - координаты правого нижнего угла
- **Viewport**: размер окна браузера

### История:
- **History: N** - количество записанных изменений (до 100 последних)

## 🎬 Типы действий в истории

1. **`init`** - Инициализация debug панели
2. **`drag`** - Перемещение окна
3. **`resize`** - Изменение размера окна
4. **`adjust`** - Автоматическая корректировка позиции (когда окно выходило за границы)

## 🛠️ Доступные действия

### 1. 📋 Copy Current
Копирует **текущее состояние** в буфер обмена:
```
Position: top: 100px, left: 200px
Size: 850×600px
Bounds: right: 1050px, bottom: 700px
Viewport: 1920×1080px
```

### 2. 💾 Export History
Скачивает **полную историю** в JSON файл с именем `modal-debug-history-{timestamp}.json`:

```json
[
  {
    "time": "2026-01-13T12:34:56.789Z",
    "action": "init",
    "position": { "top": 100, "left": 200 },
    "size": { "width": 850, "height": 600 },
    "bounds": { "right": 1050, "bottom": 700 }
  },
  {
    "time": "2026-01-13T12:35:01.234Z",
    "action": "drag",
    "position": { "top": 120, "left": 250 },
    "size": { "width": 850, "height": 600 },
    "bounds": { "right": 1100, "bottom": 720 }
  },
  {
    "time": "2026-01-13T12:35:05.678Z",
    "action": "resize",
    "position": { "top": 120, "left": 250 },
    "size": { "width": 900, "height": 650 },
    "bounds": { "right": 1150, "bottom": 770 }
  },
  {
    "time": "2026-01-13T12:35:08.901Z",
    "action": "adjust",
    "position": { "top": 120, "left": 200 },
    "size": { "width": 900, "height": 650 },
    "bounds": { "right": 1100, "bottom": 770 }
  }
]
```

### 3. 🗑️ Clear
Очищает всю историю изменений.

## 🐛 Отладка проблем

### Проблема: "Окно улетает за границу экрана"

**Шаги отладки:**

1. Включите debug панель (🔍)
2. Воспроизведите проблему:
   - Перетащите окно к краю экрана
   - Попробуйте изменить размер
3. Нажмите **💾 Export History**
4. Откройте скачанный JSON файл

**Что искать в истории:**

```json
// ❌ ПРОБЛЕМА: top изменился во время resize
{
  "time": "2026-01-13T12:35:05.000Z",
  "action": "resize",
  "position": { "top": 50, "left": 100 },  // ← top был 50
  "size": { "width": 900, "height": 650 }
},
{
  "time": "2026-01-13T12:35:05.100Z",
  "action": "resize",
  "position": { "top": -20, "left": 100 },  // ← top стал -20 (улетел!)
  "size": { "width": 920, "height": 670 }
}
```

**Ожидается:**
```json
// ✅ ПРАВИЛЬНО: top не меняется во время resize
{
  "action": "resize",
  "position": { "top": 50, "left": 100 }  // ← top стабилен
},
{
  "action": "resize",
  "position": { "top": 50, "left": 100 }  // ← top не изменился
}
```

### Проблема: "Окно прыгает после drag"

**Что искать:**

```json
// ❌ ПРОБЛЕМА: большой скачок координат
{
  "action": "drag",
  "position": { "top": 200, "left": 300 }
},
{
  "action": "drag",
  "position": { "top": 500, "left": 300 }  // ← резкий скачок на 300px
}
```

## 📈 Анализ производительности

История также помогает выявить проблемы с производительностью:

1. **Слишком частые обновления** - если записей много за короткое время
2. **Зависания** - если между записями большие временные промежутки
3. **Дублирование** - если одинаковые координаты записываются повторно

## 🔧 Технические детали

### Ограничения:
- **Максимум 100 записей** - хранятся только последние 100 изменений
- **Только при включенной панели** - история не записывается когда панель выключена
- **В памяти** - история сбрасывается при закрытии окна

### Формат временной метки:
```typescript
timestamp: 1705147496789  // Unix timestamp в миллисекундах
time: "2026-01-13T12:34:56.789Z"  // ISO 8601 в экспорте
```

## 💡 Рекомендации

### Для разработчиков:
1. **Всегда включайте debug панель** при тестировании изменений в логике drag/resize
2. **Экспортируйте историю** перед закрытием окна, если заметили баг
3. **Анализируйте action типы** - `adjust` указывает на выход за границы

### Для отчетов об ошибках:
1. Включите debug панель
2. Воспроизведите ошибку
3. Экспортируйте историю
4. Приложите JSON файл к отчету

## 🎨 Визуальные индикаторы

- **Зеленый текст** - текущие значения (можно выделить и скопировать)
- **Голубой текст** - метки полей
- **Золотой заголовок** - название панели + счетчик истории
- **Кнопки с цветами**:
  - 🔵 Синяя - Copy Current
  - 🟢 Зеленая - Export History
  - 🔴 Красная - Clear

## 🚀 Быстрый старт

```
1. Открыть VS Modal Compromise
   └─ Нажать золотую кнопку "💎 Запустить Compromise v2.0"

2. Включить debug панель
   └─ Нажать 🔍 в заголовке окна

3. Воспроизвести проблему
   └─ Drag / Resize окна

4. Экспортировать историю
   └─ Нажать "💾 Export History"

5. Анализ
   └─ Открыть JSON файл в редакторе
```

Готово! Теперь можно отслеживать все изменения координат окна. 📊
