# Тестовая страница модальных окон

**Путь:** `/tools/test-window-seamless`

## Описание

Комплексная тестовая страница для сравнения всех реализаций модальных окон в Aurora Admin.

---

## 🧪 Доступные тесты

### 1. 💎 VS Modal Compromise v2.0 (PRODUCTION)
**Статус:** ⭐ Рекомендуется для использования

**Особенности:**
- Гибридная архитектура (Portal + 3-level DI)
- Signals для реактивности (title, statusText, isResizing)
- getBoundingClientRect() для надежного resize
- Проверка границ viewport при resize
- Визуальная полировка (скроллбары, иконка, resize handle)

**Кнопка:** `💎 Запустить Compromise v2.0`

**Код:**
```typescript
openCompromiseModal(): void {
  const ref = this.vsCompromise.open(CompromiseTestContentComponent, {
    title: 'VS Modal Compromise v2.0',
    width: 850,
    height: 600,
    data: { id: 'aurora-123' },
    statusText: 'Engine Ready',
    resizable: true,
    draggable: true,
  });
}
```

---

### 2. 🟦 VS Modal (Original)
**Статус:** ✅ Стабильная версия

**Особенности:**
- ComponentPortal + cdkPortalOutlet
- Signals для динамического обновления UI
- 2-уровневая DI (может быть проблема при вложенных модалках)
- parseInt() для размеров (потенциальные баги)

**Кнопка:** `🟦 VS Modal Original`

**Путь к коду:** `src/app/shared/components/ui/vs-modal/`

---

### 3. 🟪 VS Modal Claude (Alternative)
**Статус:** 🧪 Экспериментальная

**Особенности:**
- ViewContainerRef.createComponent (императивный подход)
- 3-уровневая DI (безопаснее при вложенности)
- Статичный config (нет динамического обновления)
- getBoundingClientRect() для resize
- Улучшенная визуализация

**Кнопки:**
- `🟪 VS Modal Claude - Standard` (800x600)
- `🟪 VS Modal Claude - Large` (1200x800)
- `🟪 Claude - No Backdrop` (без затемнения)

**Путь к коду:** `src/app/shared/components/ui/vs-modal-claude/`

---

### 4. 🖼️ Aurora Image Studio
**Статус:** ✅ Production-ready

**Особенности:**
- Компонент загрузки изображений
- Встроенный редактор изображений

## 🧪 Как тестировать

### 1. Базовый тест
1. Откройте `/tools/test-window-seamless`
2. Нажмите любую кнопку
3. Проверьте открытие модального окна

### 2. Тест Drag
1. Откройте модальное окно
2. Захватите заголовок мышью
3. Перетащите окно в разные углы экрана
4. Убедитесь что окно не выходит за границы

### 3. Тест Resize
1. Откройте модальное окно
2. Захватите правый нижний угол (resize handle)
3. Измените размер окна
4. Убедитесь что окно не выходит за правую/нижнюю границу экрана

### 4. Тест Resize после Drag
1. Откройте модальное окно
2. Перетащите окно к правому краю экрана
3. Попробуйте расширить окно вправо
4. ✅ **Ожидается:** Окно не выйдет за границу (исправлено в Compromise v2.0)
5. ❌ **Баг (старые версии):** Окно уходит за границу экрана

### 5. Тест реактивности (только Compromise и Original)
1. Откройте модальное окно с тестовым контентом
2. Найдите кнопки "Update Title" или "Update Status"
3. Нажмите на них
4. ✅ **Ожидается:** Заголовок/статус обновятся мгновенно
5. ❌ **Claude версия:** Статичный config, обновление невозможно

## 🐛 Известные проблемы

### VS Modal (Original)
- ⚠️ parseInt() может давать неправильные размеры
- ⚠️ Нет проверки границ экрана при resize
- ⚠️ 2-уровневая DI может конфликтовать при вложенных модалках

### VS Modal Claude
- ⚠️ Нет Signals - невозможно динамически обновить UI
- ⚠️ ViewContainerRef вместо Portal - менее Angular-way
- ⚠️ Статичный config

### AV-Modal (Legacy)
- ❌ Устарело, не поддерживается
- ❌ Отсутствует типизация
- ❌ Нет современных фич
