## 1. Назначение модуля

Модуль управления платформами предназначен для:

- Централизованного хранения информации о поддерживаемых платформах (ОС, устройства)
- Мультиязычной локализации названий и описаний платформ
- SEO-оптимизации публичных страниц платформ
- Фильтрации и группировки приложений по платформам
- Отображения страниц "Скачать для [Платформа]"

## 2. Структура данных

### 2.1 Основная модель Platform

```typescript
interface PlatformItemDto {
  id: string; // Guid - уникальный идентификатор
  name: string; // Техническое название (для админов)
  code: string; // Системный код (уникальный, латиница)
  family?: string; // Семейство: desktop | mobile | web | console
  isActive: boolean; // Статус активности
  sortOrder: number; // Порядок сортировки при отображении
  createdAt: string; // Дата создания (ISO 8601)
  updatedAt: string; // Дата последнего обновления (ISO 8601)
  localizedName?: string; // Локализованное имя (для текущего языка)
}
```

### 2.2 Детальная модель с переводами

```typescript
interface PlatformDetailDto extends PlatformItemDto {
  translations: PlatformTranslationDto[]; // Массив переводов
}

interface PlatformTranslationDto {
  languageId: number; // ID языка
  languageCode?: string; // Код языка (ru, en, de и т.д.)
  name: string; // Локализованное название
  description?: string; // Описание на языке
  seoData?: SeoDataDto; // SEO-данные для публичных страниц
}

interface SeoDataDto {
  title?: string; // SEO заголовок
  description?: string; // SEO описание
  keywords?: string; // Ключевые слова
}
```

## 6. Бизнес-правила

### 6.1 Валидация

1. **code** (системный код):
   - Обязательное поле
   - Уникальное значение
   - Только латиница, цифры, дефис
   - Нижний регистр
   - Длина: 2-50 символов
   - Пример: `windows`, `macos`, `android`

2. **name** (техническое название):
   - Обязательное поле
   - Длина: 2-100 символов
   - Используется в админ-панели

3. **family** (семейство):
   - Допустимые значения: `desktop`, `mobile`, `web`, `console`
   - Используется для группировки и фильтрации

4. **sortOrder**:
   - Целое число ≥ 0
   - Чем меньше значение, тем выше в списке
   - По умолчанию: 999

5. **translations**:
   - Минимум 1 перевод (основной язык системы)
   - `languageId` должен существовать в системе
   - `name` - обязательное поле

### 6.2 Бизнес-логика

1. **Активность платформы:**
   - Неактивные платформы не отображаются в публичных списках
   - Неактивные платформы доступны для редактирования в админ-панели
   - При деактивации выводится предупреждение о связанных приложениях

2. **Удаление платформы:**
   - Нельзя удалить платформу, если есть связанные приложения
   - При попытке удаления выводится список связанных объектов
   - Альтернатива: деактивация вместо удаления

3. **Локализация:**
   - Если для языка нет перевода, используется `name` (техническое название)
   - При создании обязательно добавить перевод для основного языка
   - SEO-данные опциональны, заполняются для публичных страниц

## 7. API Endpoints

### 7.1 Получение списка с пагинацией

```
GET /api/v1/platforms
Query Parameters:
  - pageNumber (int, default: 1)
  - pageSize (int, default: 10)
  - searchTerm (string, optional) - поиск по name и code
  - languageId (int, optional) - фильтр по языку
  - sortBy (string, optional) - поле для сортировки
  - sortDirection ('Asc' | 'Desc', optional)

Response: PlatformPagedResponse
```

### 7.2 Получение детальной информации

```
GET /api/v1/platforms/{id}
Response: PlatformDetailDto (с переводами)
```

### 7.3 Создание

```
POST /api/v1/platforms
Body: PlatformCreateDto
Response: PlatformDetailDto
```

### 7.4 Обновление

```
PUT /api/v1/platforms/{id}
Body: PlatformUpdateDto
Response: PlatformDetailDto
```

### 7.5 Удаление

```
DELETE /api/v1/platforms/{id}
Response: 204 No Content
```

### B. Схема БД

```sql
CREATE TABLE Platforms (
  Id UNIQUEIDENTIFIER PRIMARY KEY,
  Name NVARCHAR(100) NOT NULL,
  Code NVARCHAR(50) UNIQUE NOT NULL,
  Family NVARCHAR(50),
  IsActive BIT NOT NULL DEFAULT 1,
  SortOrder INT NOT NULL DEFAULT 999,
  CreatedAt DATETIME2 NOT NULL,
  UpdatedAt DATETIME2 NOT NULL
);

CREATE TABLE PlatformTranslations (
  Id INT IDENTITY PRIMARY KEY,
  PlatformId UNIQUEIDENTIFIER NOT NULL,
  LanguageId INT NOT NULL,
  Name NVARCHAR(200) NOT NULL,
  Description NVARCHAR(MAX),
  SeoTitle NVARCHAR(200),
  SeoDescription NVARCHAR(500),
  SeoKeywords NVARCHAR(500),
  FOREIGN KEY (PlatformId) REFERENCES Platforms(Id) ON DELETE CASCADE,
  FOREIGN KEY (LanguageId) REFERENCES Languages(Id),
  UNIQUE (PlatformId, LanguageId)
);
```
