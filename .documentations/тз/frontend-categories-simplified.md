# ТЗ: Реализация модуля управления "Упрощенными Категориями" (V2)

## 1. Цель
Создать в административной панели `pr_aurora_admin` полноценный интерфейс для управления новой двухуровневой структурой категорий (Category -> Subcategory). Модуль должен быть реализован по стандарту Aurora Control с использованием Angular Signals.

## 2. Основание для разработки
*   **Backend API**: `api/v1/aggregator/categories-simplified`
*   **Архитектурный паттерн**: Manager-State-API (на базе `DeveloperOfAggregatorPage`).
*   **Дизайн**: Использование компонентов Aurora и библиотеки Ng-Zorro-Antd.

---

## 3. Функциональные требования

### 3.1. Страница управления (Manager)
*   **Путь**: `/aggregator/reference/categories-simplified`
*   **Заголовок**: "Упрощенные Категории (V2)".
*   **Gear Icon (Колесико)**: Кнопка-шестеренка рядом с заголовком. По клику раскрывает/скрывает **Блок обслуживания**.
*   **Блок обслуживания (Maintenance Block)**:
    *   `Очистить БД`: Кнопка с подтверждением (Challenge modal).
    *   `Считать данные из БД`: Принудительное обновление списка.
    *   `Перенести данные из JSON в БД`: Кнопка для запуска Seed-процесса.
*   **Поиск и фильтры**:
    *   Поле текстового поиска.
    *   Фильтр по языку (Dropdown).
    *   Переключатель **"КОРЗИНА"** (Switch): При включении отображает только удаленные записи (`isDeleted = true`).

### 3.2. Таблица данных (List)
*   **Колонки**: ID, Иконка, Код (CanonicalName), Slug, Название, Порядок, Статус (Badge), Действия.
*   **Expandable Rows**: Поддержка развертывания мастер-категории для отображения вложенной таблицы подкатегорий.
*   **Действия (4 кнопки)**:
    1.  👁 `View`: Модальное окно просмотра (Read-only).
    2.  📝 `Edit`: Модальное окно или страница редактирования.
    3.  🗑 `Soft Delete` (иконка `rest`): Перенос в корзину. Вызывает `modalService.confirm` с текстом: *"Запись будет перемещена в корзину. Её можно будет восстановить позже."*
    4.  🔥 `Hard Delete` (иконка `fire`): Полное удаление. Вызывает `modalService.challenge` (нужно решить пример, например `2+2=4`) с текстом: *"ВНИМАНИЕ: Это действие безвозвратно удалит запись из базы данных."*
*   **Восстановление**: В режиме корзины вместо кнопок удаления отображается иконка `undo` (Restore), вызывающая `modalService.confirm`.

### 3.3. Статус-бар (Footer)
*   **Sticky Status Bar**: Закрепленная панель внизу страницы.
*   **Содержимое**:
    *   `Всего записей`: Счетчик из сигнала `total`.
    *   `Статус`: Индикатор "Справочник актуален" или "Загрузка...".
    *   `Версия`: Метка версии модуля (например, v3.5.0).
