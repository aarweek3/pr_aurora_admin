# Скриншоты (Screenshots)

Система управления скриншотами в Агрегаторе программ Aurora v3.5 основана на принципе **Convention over Configuration** (Соглашение важнее конфигурации). Это позволяет массово загружать и синхронизировать изображения, просто соблюдая структуру папок и правила именования файлов.

## 1. Логика хранения (Файловая система)

Все скриншоты хранятся в директории `wwwroot` бэкенда.

**Базовый путь:**
`wwwroot/argregator_icons/programs/screenshots/`

**Структура папок:**
`{первая_буква_slug}/{slug_программы}/{код_языка}/{имя_файла}`

### Пример структуры:
```text
wwwroot/
  argregator_icons/
    programs/
      screenshots/
        v/
          visual-studio-code/
            ru/
              visual-studio-code_0.png      (Hero/Main скриншот)
              visual-studio-code_10.png     (Интерфейс)
            en/
              visual-studio-code_0.png
            common/
              visual-studio-code_100.png    (Общий скриншот, если нет языка)
```

## 2. Соглашение об именовании файлов

Файл должен начинаться со `slug` программы, за которым следует подчеркивание и приоритет.

**Шаблон:** `{slug}_{priority}.{extension}`

*   **Приоритет (SortOrder):** Число в конце имени файла. Чем меньше число, тем выше скриншот в галерее.
*   **Специальные имена:**
    *   `_hero` или `_main` -> автоматически получают `SortOrder = 0`.
    *   Если число не указано или не парсится -> `SortOrder = 100`.
*   **Язык:** Определяется по имени папки, в которой лежит файл (мапится на `ShortCode` языка в БД, например `ru`, `en`, `de`). Если папка называется `common` или не мапится на язык, скриншот считается общим (`LanguageId = null`).

## 3. Модель данных (Entity)

Скриншоты описываются сущностью `ScreenshotOfAggregator`.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `Id` | `int` | Первичный ключ. |
| `ProgramOfAggregatorId` | `int` | Связь с программой. |
| `FilePath` | `string` | Относительный путь к файлу (например, `argregator_icons/...`). |
| `SortOrder` | `int` | Порядок сортировки (извлекается из имени файла). |
| `LanguageOfAggregatorId`| `int?`| Привязка к языку (извлекается из имени папки). |

## 4. Методы и Синхронизация

### Бэкенд (C#)
Основная логика находится в `ProgramOfAggregatorService`.

*   **Метод:** `SyncScreenshotsByConventionAsync()`
*   **Логика работы:**
    1.  Сканирует папку `wwwroot/argregator_icons/programs/screenshots/`.
    2.  Для каждой программы ищет соответствующие файлы во всех подпапках языков.
    3.  **Добавление:** Если файла нет в БД, создает запись.
    4.  **Обновление:** Если файл есть, но изменился приоритет или язык (папка), обновляет запись.
    5.  **Очистка:** Если запись в БД есть, а файла на диске нет — удаляет запись из БД.

### API (Controller)
*   **Endpoint:** `POST /api/ProgramOfAggregator/sync-screenshots`
*   **Доступ:** Кнопка "Синхронизировать скриншоты" в админ-панели (блок обслуживания в хедере менеджера программ).

## 5. Использование в UI (Angular)

В интерфейсе детализации программы скриншоты группируются по языкам.

*   **Компонент:** `program-of-aggregator-details.component.ts`
*   **Метод фильтрации:** `getScreenshotsForLang(langId: number | null)`
*   **Отображение:** Скриншоты выводятся в табах соответствующих языков. Если у скриншота нет привязки к языку (`LanguageId == null`), он отображается во вкладке "Общие / Все".

## 6. Резюме для разработчика

Чтобы добавить скриншоты для программы:
1.  Узнайте `slug` программы (например, `telegram`).
2.  Создайте папку `wwwroot/argregator_icons/programs/screenshots/t/telegram/ru/`.
3.  Положите туда файлы `telegram_0.png`, `telegram_1.png`.
4.  Зайдите в админку -> Агрегатор -> Обслуживание (иконка шестеренки в заголовке).
5.  Нажмите **"Синхронизировать скриншоты"**.
6.  Система автоматически создаст записи в БД, распарсит приоритеты и привяжет их к русскому языку.
