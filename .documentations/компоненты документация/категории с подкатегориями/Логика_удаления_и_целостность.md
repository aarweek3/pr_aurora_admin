# Логика удаления и целостность данных (V2)

В модуле «Упрощенные категории» реализована сложная система удаления, которая гарантирует, что данные программ (`Programs`) никогда не будут потеряны или повреждены при манипуляциях со справочником категорий.

---

## 1. Мягкое удаление (Soft Delete)

Это основной способ удаления в административной панели. Запись не исчезает из базы данных, а помечается флагом `IsDeleted`.

### Процесс (Категория):
1. **Проверка**: Система проверяет, не является ли категория системной (например, `CAT_UNCATEGORIZED`). Системные записи удалять запрещено.
2. **Метка**: Категории и всем её вложенным подкатегориям присваивается статус `IsDeleted = true`.
3. **Миграция программ**: Все программы, привязанные к удаляемой категории или её подкатегориям, **автоматически переносятся** в системную категорию `CAT_UNCATEGORIZED`. Поле `SubcategoryId` у этих программ обнуляется (`NULL`).
4. **Видимость**: Запись перестает отображаться в общем списке, но её можно увидеть, включив фильтр «Показать удаленные» в интерфейсе.

### Процесс (Подкатегория):
1. **Метка**: Только выбранной подкатегории присваивается статус `IsDeleted = true`.
2. **Миграция программ**: Программы, которые были привязаны к этой подкатегории, переносятся в «Мастер-категорию» `UNCATEGORIZED`. Связь с конкретной подкатегорией разрывается.

---

## 2. Жесткое удаление (Hard Delete / Fire)

Это необратимое действие, которое полностью стирает записи из таблиц базы данных.

### Меры безопасности:
* **Подтверждение**: Требуется решение математического примера и ввод кодового слова «УДАЛИТЬ».
* **Цветовая индикация**: Интерфейс окрашивается в глубокий красный цвет (`modal-danger`), сигнализируя о критической операции.

### Процесс:
1. **SQL-миграция**: Перед удалением выполняется прямой SQL-запрос, который переводит все связанные программы в категорию `UNCATEGORIZED`. Это гарантирует, что у программ не останется битых ссылок (Foreign Key errors).
2. **Каскадное удаление**:
   - Сначала удаляются все локализации (переводы) из таблицы `category_localizations_simplified`.
   - Затем удаляются подкатегории и их переводы.
   - В последнюю очередь удаляется сама запись из `categories_simplified`.
3. **Результат**: Данные полностью стерты. Восстановление средствами приложения невозможно.

---

## 3. Влияние на Программы (Programs Of Aggregator)

Программы являются «потребителями» категорий. Архитектура V2 исключает ситуацию «сиротства» программ.

| Действие | Результат для Program | Почему так? |
|----------|----------------------|-------------|
| Удаление категории (Soft) | CategoryId = UNCATEGORIZED, SubcategoryId = NULL | Чтобы программа осталась доступна в админке для переназначения. |
| Удаление подкатегории | CategoryId = UNCATEGORIZED, SubcategoryId = NULL | Мы сбрасываем программу на самый верхний "безопасный" уровень. |
| Восстановление категории | Состояние Program **НЕ МЕНЯЕТСЯ** | Автоматический возврат может быть нежелателен, если за время удаления программы были переназначены вручную. |

---

## 4. Восстановление данных (Restore)

Функция восстановления доступна только для записей, удаленных «мягким» способом.

* **Категория**: При восстановлении мастер-категории её подкатегории остаются в корзине (их нужно восстанавливать отдельно, если это требуется).
* **Связи**: После восстановления категория снова появляется в выпадающих списках при редактировании программ.

---
**Важно**: Всегда проверяйте наличие системной категории `CAT_UNCATEGORIZED` в базе данных. Она является «страховочной сеткой» для всего модуля.
