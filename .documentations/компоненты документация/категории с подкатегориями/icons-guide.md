# Документация: Управление иконками категорий (Simplified Categories)

Система иконок в модуле упрощенных категорий спроектирована как гибридное решение, объединяющее кастомные SVG-иконки сервера, внешние ассеты и системную библиотеку Ant Design.

## 1. Архитектура хранения

Иконки управляются централизованно на стороне бэкенда через JSON-файл:
- **Путь на сервере**: `Project_Server_Auth/Pages/AGGREGATOR/CategorySimplified/Jsons/category-icons.json`
- **Формат данных**:
```json
[
  {
    "name": "av-games",
    "svgCodIcon": "<svg>...</svg>"
  }
]
```

При запуске модуля фронтенд загружает весь список иконок одним запросом через эндпоинт `GET /api/v1/aggregator/categories-simplified/icons`.

## 2. Поддерживаемые форматы (Приоритеты)

Система отображения иконок работает по принципу «fallback» в следующем порядке:

1.  **Custom SVG (JSON)**: Если значение в поле `IconUrl` совпадает с `name` из JSON-файла, рендерится SVG-код.
2.  **Asset/External URL**: Если строка содержит символы пути (`/`) или расширения (`.`), она трактуется как ссылка на файл (PNG, SVG, WebP). Используется `ImageServiceUniversal` для формирования полного пути.
3.  **Ant Design Icon**: Если строка является простым словом (например, `folder`), система пытается найти иконку в стандартной библиотеке Ng-Zorro (`nz-icon`).
4.  **Default**: Если поле пустое, отображается системная иконка папки.

## 3. Интерфейс управления (UI)

В модальных окнах создания/редактирования категории и подкатегории реализован **Icon Picker**:
- Поле «Иконка» представлено выпадающим списком с поиском.
- Группа **«Кастомные (SVG)»**: подгружается автоматически из JSON-файла с сервера.
- Группа **«Системные (Ant Design)»**: содержит базовые варианты (folder, appstore и т.д.).
- Список поддерживает **предпросмотр** (превью) иконки рядом с её названием.

## 4. Инструкция: Как добавить новую иконку

Чтобы добавить новую иконку в систему:

1.  Подготовьте SVG-код иконки (рекомендуется размер 24x24 или 32x32, `fill="none" stroke="currentColor"` для поддержки системных цветов).
2.  Откройте файл `category-icons.json` на бэкенде.
3.  Добавьте новый объект в массив:
    ```json
    {
      "name": "my-new-icon",
      "svgCodIcon": "<svg>...</svg>"
    }
    ```
4.  Обновите страницу в админке. Новая иконка появится в выпадающем списке модального окна.
5.  Выберите её и нажмите «Сохранить».

## 5. Технические детали реализации

- **Sanitization**: На фронтенде используется `DomSanitizer.bypassSecurityTrustHtml` для безопасного рендеринга SVG из JSON.
- **Стилизация**: Кастомные SVG оборачиваются в контейнер `.custom-svg-icon`, который автоматически подстраивает размер и цвет иконки под системную тему Aurora.
- **Производительность**: Иконки кэшируются в `CategorySimplifiedStateService` на всё время сессии работы с модулем.
