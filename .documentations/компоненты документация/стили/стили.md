# Стандарт интерфейса Aurora (UI/UX Standard)

Этот документ описывает правила работы со стилями, модальными окнами и кнопками в проекте. Соблюдение этих стандартов гарантирует "премиальный" вид интерфейса и простоту поддержки.

## 1. Дизайн-токены (CSS Переменные)

Все ключевые параметры интерфейса вынесены в файл `src/styles/abstracts/_tokens.scss`. 
**Никогда не используйте жесткие значения (пиксели) в компонентах, если есть соответствующий токен.**

### Модальные окна (Modal System)
*   `--av-modal-width-small`: **600px** (Подтверждения, алерты).
*   `--av-modal-width-medium`: **720px** (Стандартные формы).
*   `--av-modal-width-large`: **1000px** (Таблицы, сложные данные).
*   `--av-modal-radius`: **12px** (Скругление всех панелей).
*   `--av-modal-backdrop-blur`: **4px** (Размытие фона).

---

## 2. Глобальные премиум-кнопки

Для модальных окон и важных интерфейсов используются специальные "премиальные" классы, заданные в `_modal.scss`. Они имеют встроенные анимации, тени и профессионально подобранные цвета.

| Класс | Цвет (Base) | Hover Эффект | Когда использовать |
|-------|------------|--------------|-------------------|
| `.btn-primary-premium` | #1890ff (Blue) | Светлее + тень | Основное действие (ОК, Сохранить, Создать) |
| `.btn-danger-premium` | #b91c1c (Deep Red) | Насыщенный красный + свечение | Удаление, деструктивные действия |
| `.btn-cancel-premium` | Белый/Серый | Светло-голубой фон | Отмена, закрытие без сохранения |

**Пример использования в HTML:**
```html
<div class="footer-actions">
  <button class="btn-cancel-premium">Отмена</button>
  <button class="btn-primary-premium">Сохранить изменения</button>
</div>
```

## 3. Модальные окна (Modal Management)

### Класс `.modal-danger`
Если вы открываете окно с критическим действием, передайте в конфигурацию `panelClass: 'modal-danger'`.
Это автоматически окрасит заголовок окна в глубокий красный цвет (`#b91c1c`).

### Закрытие (X)
Кнопка закрытия в углу модального окна автоматически настроена:
*   Тонкая линия (opacity 0.6).
*   Анимация поворота и покраснения при наведении.

---

## 4. Работа с иконками (`av-icon`)

В проекте реализована гибридная система загрузки иконок через `IconGetService`.

1.  **Локальные иконки (System/Actions)**:
    Если в типе иконки указан путь со слешем (например, `actions/av_trash`), система **автоматически** ищет файл в `src/assets/icons/actions/av_trash.svg`.
    *Это предпочтительный способ для всех иконок интерфейса.*

2.  **API иконки**:
    Если указано просто имя без пути (например, `folder`), система запрашивает иконку через API (или использует шрифт Ant Design).

---

## 5. Психология цветов в проекте

*   **Красный (`#b91c1c`)**: Не используйте яркий "кислотный" красный. Используйте глубокий Crimson для ощущения серьезности.
*   **Синий (`#1890ff`)**: Основной цвет бренда для позитивных действий.
*   **Голубой (`#eff6ff`)**: Цвет "безопасной гавани". Используется для фокуса и наведения на кнопки отмены.
