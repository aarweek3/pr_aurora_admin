# Документация: Работа с видео в карточке программы

В системе реализована гибкая двухслойная структура для управления видео-контентом. Это позволяет задавать общие ролики для всех регионов и специфические видео для конкретных локализаций.

## 1. Структура хранения

### Глобальные видео (Шаг 2)
Находятся в основной таблице программы. Используются как **fallback** (резервный вариант), если для выбранного пользователем языка не задано локальное видео.
*   **YoutubeVideoUrl**: Общая ссылка на YouTube.
*   **CustomVideoUrl**: Ссылка на файл для внутреннего плеера.

### Локализованные видео (Шаг 3)
Находятся во вкладках языков. Позволяют показывать контент на языке пользователя (например, обзор на русском для RU-локации).
*   **YoutubeVideoUrl**: Ссылка на YouTube именно для этого языка.
*   **CustomVideoUrl**: Ссылка на файл именно для этого языка.

---

## 2. Форматирование YouTube ссылок

Система ожидает стандартные ссылки. Для корректного встраивания (через iframe) рекомендуется использовать следующие форматы:

### Рекомендуемый формат (Browser URL)
Просто скопируйте ссылку из адресной строки браузера:
`https://www.youtube.com/watch?v=dQw4w9WgXcQ`

### Короткий формат (Share)
Ссылка из кнопки "Поделиться":
`https://youtu.be/dQw4w9WgXcQ`

### Формат для встраивания (Embed)
Если вы копируете код вставки:
`https://www.youtube.com/embed/dQw4w9WgXcQ`

### YouTube Shorts
Ссылки на короткие вертикальные видео также поддерживаются:
`https://www.youtube.com/shorts/RANDOM_ID`

> [!TIP]
> Фронтенд-компонент автоматически извлекает ID видео из любого из вышеперечисленных форматов для формирования корректного плеера.

---

## 3. Custom Video (Наш плеер)

Поле **Custom Video URL** предназначено для прямых ссылок на видеофайлы, которые будут проигрываться встроенным плеером системы (не YouTube).

**Поддерживаемые форматы:**
*   **MP4**: Самый универсальный формат (`.mp4`).
*   **WebM**: Оптимизированный формат для браузеров (`.webm`).
*   **HLS (m3u8)**: Потоковое видео для адаптивного качества (`.m3u8`).

---

## 4. Логика отображения (Frontend)

При отображении видео в публичной части сайта используется следующий приоритет:

1.  **Локальное видео**: Если у текущей локализации (например, RU) заполнено поле видео — показываем его.
2.  **Глобальное видео**: Если локальное поле пустое, берем ссылку из основной карточки программы.
3.  **Скрытие**: Если ни одно поле не заполнено, блок видео в карточке программы не отображается.

---

## 5. Рекомендации для администраторов

1.  Если видео универсальное (геймплей без слов), заполняйте только **Шаг 2**.
2.  Если есть качественный обзор на конкретном языке — обязательно добавьте его в **Шаг 3** для соответствующей локализации.
3.  Для YouTube всегда старайтесь использовать ссылки `https://www.youtube.com/watch?v=...`, так как они наиболее надежны для парсинга.
