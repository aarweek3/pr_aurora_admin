# Пошаговая инструкция по созданию нового модуля

Следуйте этому чек-листу для создания нового полнофункционального модуля в проекте Aurora Admin.

## Шаг 1: Подготовка структуры
1. Создайте корневую папку модуля (например, в `src/app/pages/` или `src/assets/`).
2. Внутри создайте папки: `components`, `services`, `models`, `config`.
3. Создайте файл `end-points.ts` в корне модуля.

## Шаг 2: Определение модели данных (`models/`)
1. Создайте интерфейс основной сущности (например, `User.model.ts`).
2. Опишите интерфейс состояния модуля (например, `UserState` с полями `items`, `isLoading` и т.д.).

## Шаг 3: Настройка связи с API
1. Пропишите пути в `end-points.ts`.
2. Создайте `*-api.service.ts` в папке `services/`.
3. Реализуйте методы `get`, `post`, `put`, `delete`.

## Шаг 4: Реализация бизнес-логики (`services/`)
1. Создайте основной сервис модуля (например, `User.service.ts`).
2. Инициализируйте `signal` для состояния.
3. Реализуйте публичные методы для обновления состояния (вызывая `apiService`).
4. Используйте `computed` для предоставления данных компонентам.

## Шаг 5: Разработка UI-компонентов (`components/`)
1. Создайте основной менеджер-компонент (например, `user-manager.component.ts`).
2. Используйте `standalone: true`.
3. Внедрите (`inject`) ваш бизнес-сервис.
4. Используйте `NzTableModule` или другие компоненты Ng-Zorro для отображения данных.
5. При необходимости создайте вспомогательные компоненты для форм и модальных окон.

## Шаг 6: Интеграция
1. Добавьте маршруты (routes) для нового компонента в `app.routes.ts` или в локальный файл роутинга.
2. Добавьте ссылку на новый модуль в конфигурацию сайдбара (`sidebar-default.config.ts`).

---

### Полезные команды (Angular CLI):
```bash
# Создание компонента
ng g c pages/my-module/components/my-component --standalone

# Создание сервиса
ng g s pages/my-module/services/my-service
```
