Интеграция TinyMCE в Aurora Admin

В проекте Aurora используется кастомная интеграция TinyMCE 8.3.1, развернутая как локальный статичный ресурс. Это позволяет полностью контролировать плагины, локализацию и стили без зависимости от внешних CDN.

1. Архитектура подключения

Реализация разделена на три уровня:

  • Static Assets: Ядро редактора находится в src/assets/tinymce/.
  • Angular Control: Компонент-обертка AvTinymceControlComponent (расположен в src/app/shared/controls/tinymce-control/tinymce-control.component.ts).
  • Bridge Service: Сервис TinymceBridgeService, обеспечивающий связь между TinyMCE и Angular (расположен в src/app/shared/services/tinymce-bridge.service.ts).

2. Конфигурация компонента (Control)

Компонент AvTinymceControlComponent реализует ControlValueAccessor, что позволяет использовать его в Angular Reactive Forms и через [(ngModel)].

Ключевые настройки инициализации: 
tinymce.init({
  base_url: '/assets/tinymce', // Путь к статике
  suffix: '.min',
  license_key: 'gpl',          // Используется GPL версия
  language: 'ru',              // Локализация из src/assets/tinymce/langs/ru.js
  plugins: this.plugins,       // Список стандартных и кастомных плагинов
  external_plugins: {          // Подключение внешних скриптов плагинов
    'av-image': '/assets/tinymce/plugins/av-image/plugin.js',
    ...
  }
});

3. Кастомное брендирование и UI

Для создания единого стиля "Aurora" переопределены стандартные иконки TinyMCE и добавлены брендированные элементы:

  • Иконка Aurora: Зарегистрирована под именем av_image_aurora.
  • Кнопка Studio: Добавлена кнопка av-image-studio на панель инструментов, которая через Bridge открывает Angular-редактор.
  • Переопределение Media: Иконки embed и media заменены на кастомные зеленые SVG в стиле Aurora.

4. Механизм Bridge (Мост)

Поскольку TinyMCE работает вне зоны (NgZone) Angular, для вызова Angular-сервисов и открытия компонентов используется TinymceBridgeService.

При нажатии на кнопку в TinyMCE:

  1. Вызывается метод bridgeService.openImageStudio(editor).
  2. Сервис принудительно входит в ngZone.run().
  3. Открывается модальное окно через VSModalService.
  4. После закрытия окна результат (например, HTML-код изображения) вставляется обратно в редактор командой editor.insertContent().

5. Безопасность и валидация

Для корректной работы сложной разметки (фигуры, подписи, обертки) в настройках расширены правила валидации HTML: 

extended_valid_elements: 'figure[*],figcaption[*],img[*],a[*]',
valid_children: '+figure[figcaption|img|a],+a[figure|img]'
