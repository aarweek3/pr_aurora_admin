🖥️ Авторизация: Backend (ASP.NET Core)

Бэкенд Aurora базируется на Microsoft Identity и собственной системе управления JWT-сессиями в куках, что позволяет избежать хранения токенов в LocalStorage фронтенда.

📂 Ключевые компоненты

AuthController.cs Входные точки (Login, Register, Logout, Refresh). Работа с Response.Cookies.
AuthService.cs Бизнес-логика: проверка паролей через UserManager, генерация токенов, управление сессиями в БД.
TokenService.cs Генерация JWT строк, формирование Claims (ID пользователя, Роли, Email).
IdentityExtensions.cs Методы расширения для регистрации сервисов Identity и JWT в IServiceCollection.

📊 Модели данных (Entity Framework)

// Наследуется от IdentityUser
public class AppUser : IdentityUser 
{
    public string? AvatarUrl { get; set; }
    public DateTime CreatedAt { get; set; }
    public virtual ICollection<UserSession> Sessions { get; set; }
}

// Хранение активных Refresh-токенов
public class UserSession 
{
    public int Id { get; set; }
    public string UserId { get; set; }
    public string RefreshToken { get; set; }
    public DateTime ExpiresAt { get; set; }
    public bool IsRevoked { get; set; }
}

🛡 Безопасность Cookies

При логине сервер отправляет заголовок Set-Cookie со следующими параметрами:

  • HttpOnly: true (запрет доступа из JS).
  • Secure: true (только через HTTPS).
  • SameSite: Strict или Lax (защита от CSRF).

💡 Принцип работы: Сервер проверяет входящую куку accessToken в каждом запросе через стандартный Middleware. Если она истекла, но есть refreshToken, фронтенд инициирует обновление.
