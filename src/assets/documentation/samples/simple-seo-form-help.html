Сравнение моделей: Simple vs Basic vs SEO

В архитектуре Aurora Admin предусмотрено три основных шаблона для работы с данными разной сложности. От простых справочников до SEO-оптимизированного контента. Это ruководство поможет вам выбрать оптимальную структуру для новой сущности.

1. Сводная таблица сравнения всех трех моделей

Характеристика 📦 Sample Simple 🌐 Sample Языки (Basic) 🚀 Sample + Языки + SEO
Назначение Простые справочники без многоязычности Служебные справочники с поддержкой языков Публичный контент (статьи, категории, товары)
Многоязычность ❌ Не поддерживается ✅ Полная поддержка (selector + языковые данные) ✅ Вкладки по языкам в форме
SEO-инструменты ❌ Отсутствуют ❌ Отсутствуют ✅ Полный набор (Meta, OG, Schema.org)
Интерфейс формы Простое модальное окно Модальное окно (выбор языка за сеанс) Единая форма с вкладками всех языков
Режимы хостинга Modal Modal Modal, Inline (Split), Full Page
Медиа-данные ❌ Только текст ❌ Только текст ✅ Изображения, флаги, локализованные медиа
Количество компонентов 5 (Manager, Table, Modal + 2 сервиса) 6 (Manager, Table, Modal + 2 сервиса + подбор языков) 8-10 (+ SEO-форма, + вкладки, + медиа)
Сложность ⭐ Самая простая ⭐⭐ Средняя ⭐⭐⭐ Сложная
Время разработки 1-2 дня 2-3 дня 5-7 дней

2. Рекомендации по выбору

📦 Выбирайте Sample Simple для:

  • Общих справочников в админ-панели
  • Внутренних данных без публичного отображения
  • Простых CRUD операций
  • Когда не нужна многоязычность
  • Быстрого прототипирования
Примеры: Должности, Регионы, Типы

🌐 Выбирайте Sample Языки для:

  • Справочников с поддержкой языков
  • Многоязычного контента
  • Служебных данных (не для публики)
  • Когда нужна локализация
  • Отсутствуют SEO требования
Примеры: Теги (тегированные на разных языках), Системные константы

🚀 Выбирайте Sample + SEO для:

  • Публичного контента
  • SEO-оптимизированного контента
  • Индексирования поисковиками
  • Многоязычного контента с SEO
  • Привлечения органического трафика
Примеры: Статьи, Категории товаров, Лендинги, Бренды

3. Как это реализовано в Справке

Каждая страница имеет кнопку Помощь (?) в правом верхнем углу, которая ведет на данный сравнительный анализ.

4. Архитектура Sample Simple (📦 Простая модель)

Самая базовая архитектура, состоит из 5 элементов без поддержки многоязычности:

Компонент / Сервис Описание
SimpleManagerComponent Контейнер с поиском, пагинацией, кнопкой добавления. Управляет жизненным циклом.
SimpleTableComponent Таблица отображения (view mode). Получает @Input данные, генерирует @Output события.
SimpleModalComponent Форма в модальном окне. Режимы: ADD, EDIT, VIEW. Реактивные формы.
SampleSimpleService BehaviorSubject для управления состоянием. Методы: search(), changePage(), openAdd(), save().
SampleSimpleApiService HTTP запросы к API. Методы: getSamples(), getById(), create(), update(), delete().

5. Архитектура Sample Языки (🌐 Basic Model)

Расширение Simple с поддержкой многоязычности и языковыми селекторами:

Компонент / Сервис Описание
SimpleMainManagerComponent + Селектор языков. Фильтрация списка по выбранному языку. Управление состоянием.
SimpleMainTableComponent Таблица с колонкой "Локализованное имя". Отображает данные выбранного языка.
SimpleMainModalComponent Форма с language selector. Редактирование для одного языка за раз.
SampleMainStateService + setLanguage(). State: {items, languageId, searchTerm, pageNumber, ...}
SampleMainApiService + параметр languageId в getPaged(). + методы для многоязычных данных.
LanguageService (фреймворк) Услуга из Aurora фреймворка для получения списка доступных языков.

6. Архитектура Sample + SEO (🚀 Продвинутая модель)

Полная SEO-оптимизация с многоязычностью, вкладками и медиа-поддержкой:

Компонент / Сервис Описание
SampleMainSeoManagerComponent Расширенный контейнер. Может работать в 3 режимах: Modal, Inline (Split), Full Page.
SampleMainSeoTableComponent Таблица с доп. информацией: Meta Title preview, Status flags.
SampleMainSeoFormComponent Вкладки по языкам. Все языки редактируются одновременно. Включает SEO-поля.
SeoFormComponent (shared) Переиспользуемый компонент для SEO-полей (Meta Title, Meta Desc, OG, Schema.org).
SampleMainSeoStateService Расширенное управление состоянием. Поддержка draft/published статусов.
SampleMainSeoApiService Запросы к другому endpoint: /api/sample-main-seo/paged. + методы для медиа.
MediaUploadService Загрузка и управление локализованными изображениями и флагами.

7. Поток данных (Data Flow) для каждой модели

7.1 Sample Simple: Простой поиск

input → SimpleManagerComponent.onSearch() → SampleSimpleService.search()
→ SampleSimpleApiService.getSamples() → SimpleTableComponent (через async pipe)

7.2 Sample Языки: Поиск с фильтром по языку

Селектор языка → SimpleMainManager.onLanguageChange() → SampleMainStateService.setLanguage(langId)
→ updateState({languageId}) → API getPaged({languageId}) → Таблица отображает локализованные данные

7.3 Sample + SEO: Редактирование всех языков одновременно

Редактировать (кнопка) → SampleMainSeoManagerComponent.openEdit(id)
→ SampleMainSeoFormComponent открывается с 3+ вкладками (один язык = одна вкладка)
→ Все SEO-поля видны одновременно для всех языков
→ Сохранение отправляет multipart request на API с данными всех языков

8. Особенности и отличия трех моделей

📦 Sample Simple — Минимализм

  • ✅ Самая быстрая в разработке (1-2 дня)
  • ✅ Минимум компонентов (5)
  • ✅ Простее дебажить
  • ✅ Подходит для внутренних справочников
  • ❌ Нет многоязычности
  • ❌ Нет SEO
  • ❌ Не подходит для публичного контента

🌐 Sample Языки (Basic) — Универсал

  • ✅ Поддержка многоязычности
  • ✅ Хороший баланс с Simple (шаблон можно переиспользовать)
  • ✅ Подходит для служебных справочников на разных языках
  • ✅ Гибкий API с languageId фильтром
  • ⚠️ Одновременно можно редактировать только один язык
  • ❌ Нет SEO-полей
  • ❌ Нет поддержки локализованных медиа

🚀 Sample + SEO (Продвинутая) — Мощь

  • ✅ Полный набор SEO-инструментов
  • ✅ Редактирование всех языков одновременно
  • ✅ Поддержка локализованных изображений
  • ✅ 3 режима: Modal, Inline, Full Page
  • ✅ Публикация статусов (Draft/Published)
  • ✅ Meta Title preview в таблице
  • ⚠️ Сложнее разработка (5-7 дней)
  • ⚠️ Требует опыта работы с RxJS и Reactive Forms

7. Примеры кода

7.1 Вызов поиска в Manager Component

// SimpleMainManagerComponent
onSearch(event: Event): void {
  const term = (event.target as HTMLInputElement).value;
  this.service.search(term);
}

7.2 State Service управляет состоянием

// SampleMainStateService
private state$ = new BehaviorSubject<SampleMainState>(INITIAL_STATE);

getState(): Observable<SampleMainState> {
  return this.state$.asObservable();
}

search(term: string): void {
  this.updateState({ searchTerm: term, pageNumber: 1 });
  this.loadItems();
}

7.3 Подписка на состояние в компоненте

// В SimpleMainManagerComponent
context: { service: SampleMainStateService } = inject(...);
state$ = this.service.getState();

// В template
<app-sample-main-table
  [items]="(state$ | async)?.items || []"
  [loading]="(state$ | async)?.loading || false"
></app-sample-main-table>

7.4 API Service вызывает backend

// SampleMainApiService
getPaged(params: any): Observable<{items, total}> {
  return this.http.get('/api/sample-main/paged', { params });
}

getById(id: number): Observable<SampleMainDetailDto> {
  return this.http.get(`/api/sample-main/{id}`);
}

9. Примеры кода трех моделей

9.1 Sample Simple: Поиск без многоязычности

// Без languageId - просто поиск по названию
this.api.getSamples({
  pageNumber: 1,
  pageSize: 10,
  searchTerm: 'test',
  // languageId отсутствует!
});

9.2 Sample Языки: Поиск с languageId

// С languageId - фильтрация по языку
this.api.getPaged({
  pageNumber: 1,
  pageSize: 10,
  searchTerm: 'test',
  languageId: 2, // Русский
  sortBy: 'Id',
});

9.3 Sample + SEO: Multi-Language Form

// Form с вкладками для всех языков
<nz-tabset>
  <nz-tab *ngFor="let lang of languages" [nzTitle]="lang.title">
    <app-seo-form
      [formGroup]="getLanguageForm(lang.id)"
      [languageId]="lang.id"
    ></app-seo-form>
  </nz-tab>
</nz-tabset>

11. ✅ Чек-лист разработчика по выбору модели

Вопросы для определения нужной модели:

Вопрос 1: Нужна ли многоязычность (поддержка 2+ языков)?
✓ ДА → идите на Basic или SEO
✗ НЕТ → Sample Simple (конец)
Вопрос 2: Будет ли контент публичным и нужна ли SEO?
✓ ДА → Sample + SEO (продвинутая)
✗ НЕТ → Sample Языки (Basic)
Вопрос 3: Нужна ли поддержка медиа (изображения, флаги)?
✓ ДА → Sample + SEO (обязательно)
✗ НЕТ → Может быть Basic, может быть SEO (по вкусу)

Результат:

  • Sample Simple: Внутренние справочники без локализации (должности, регионы, типы)
  • Sample Языки: Служебные справочники с локализацией (теги, системные константы)
  • Sample + SEO: Публичный контент (статьи, категории товаров, лендинги, бренды)
