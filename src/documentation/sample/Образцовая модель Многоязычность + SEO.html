Образцовая модель: Многоязычность + SEO

Этот модуль является эталонным примером реализации универсального компонента в Aurora Admin. Он демонстрирует, как одна и та же форма может использоваться в разных режимах и поддерживать сложную вложенную структуру данных (SEO + Переводы).

1. Три режима хостинга формы

Архитектура компонента позволяет переключать режим отображения без изменения бизнес-логики:

  • Модальное окно (Modal): Быстрое редактирование "поверх" списка. Идеально для небольших правок.
  • Встроенная панель (Inline): Сплит-режим (список слева, форма справа). Удобно для последовательной обработки множества записей.
  • Отдельная страница (Page): Полноэкранный режим с собственным URL. Подходит для сложных записей с большим объемом контента.

2. Структура многоязычных данных

Для каждого поддерживаемого языка в системе генерируется отдельная вкладка. Внутри вкладки данные разделены на две группы:

  1. Контентные поля: Локализованное название, описание и HTML-текст.
  2. SEO-панель: Скрываемая секция, содержащая все необходимые для продвижения метаданные.

3. SEO Метаданные (на каждый язык)

Доступные поля оптимизации:
  • Meta Title и Meta Description
  • Open Graph (OG): Заголовки и изображения для социальных сетей.
  • Canonical URL
  • Schema.org: Структурированные данные в формате JSON-LD.

4. Техническая реализация

  • State Management: SampleMainSeoStateService управляет всеми операциями (Loading, Saving, Resetting).
  • API Integration: SampleMainSeoApiService обеспечивает связь с контроллером бэкенда.
  • Shared Form: SampleMainSeoFormComponent — это "чистый" компонент формы, который ничего не знает о том, где он находится (в моделе или на странице).

⚠️ Критическое требование

Перед началом работы убедитесь, что в системе **инициализированы языки**. Если список языков пуст, вкладки формы не будут созданы, что приведет к ошибкам при сохранении.

5. Сравнение с Test Sample Multi-Lang

В системе присутствуют две образцовые реализации многоязычности. Выбор между ними зависит от бизнес-задач сущности:

Характеристика Test Sample Multi-Lang Образцовая модель (SEO)
Назначение Простые справочники, внутренние данные. Контент для сайта, категории, статьи, товары.
SEO-панель Отсутствует. Полный набор (Meta, OG, Schema, Canonical).
Режимы формы Только модальное окно. 3 режима: Модалка, Встроенная, Страница.
Сложность кода Низкая (базовый CRUD). Средняя (сложное состояние, вложенные формы).

💡 Рекомендации Antigravity:

  • Используйте Test Sample Multi-Lang, если создаете служебный справочник (например, "Типы доставки" или "Статусы заказов"), где пользователю не нужно думать о продвижении в Google.
  • Используйте Образцовую модель (SEO) для любых сущностей, которые будут иметь свою страницу на публичном сайте. Это гарантирует, что контент-менеджер не забудет заполнить мета-теги, и страница будет выглядеть привлекательно в социальных сетях.

Валидация и проверки

  • Уникальность SystemCode: Проверяется на лету через асинхронный валидатор (Backend API).
  • Обязательные переводы: Система требует заполнения данных хотя бы для одного основного языка (обычно ID=1).
  • Оптимизация изображений: Рекомендуется использовать относительные пути, которые корректно обрабатываются API загрузки.
