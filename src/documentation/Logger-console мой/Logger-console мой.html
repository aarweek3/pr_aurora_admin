Logger Console 🛠️

Интеллектуальная система отладки для платформы Aurora Admin. Обеспечивает прозрачность данных, мониторинг сетевой активности и управление состоянием приложения в реальном времени.

1. Управление подключениями

Для корректной работы логгера необходимо обеспечить сбор данных (Backend) и отображение интерфейса (Frontend).

📡 HTTP Interceptor

Auto

Перехватывает все сетевые запросы. Логирует URL, время выполнения и тело ответа.

Config: LoggerInterceptor

⚡ Signal Tracking

Effect

Реактивное отслеживание изменений Signals. Сравнивает старые и новые значения.

Method: trackSignal()

👣 Breadcrumbs

Global

Автоматическая запись пользовательских кликов и навигации по всему приложению.

Type: USER_ACTION

🖥️ UI Viewport

Required

Размещение визуальной панели консоли. Должна быть в глобальном лейауте.

Tag: <app-logger-console>

2. Инициализация в приложении

Шаг 1: Регистрация компонента (Глобально)

Для того чтобы консоль отображалась, добавьте её компонент в AdminLayoutComponent или AppComponent:

<!-- admin-layout.component.html -->
<div class="admin-wrapper">
    <router-outlet></router-outlet>
    <!-- Панель консоли, обычно скрыта и вызывается кнопкой/хоткеем -->
    <app-logger-console></app-logger-console>
</div>

Шаг 2: Регистрация Интерцептора

В файле app.config.ts убедитесь, что интерцептор активен:

{ provide: HTTP_INTERCEPTORS, useClass: LoggerInterceptor, multi: true }

3. Описание функций (API)

LoggerConsoleService (Root API)

Метод Описание
getLogger(source: string | object) Создает инстанс логгера для компонента (авто-определение имени класса).
trackSignal(signal, name) Привязывает логгер к сигналу. Логирует каждое изменение.
exportToJSON() Экспорт всей накопленной истории в файл JSON.
clear() Полная очистка списков логов и команд терминала.
executeCommand(line) Выполнение команд терминала через код (напр. 'toggle-theme').

Logger Instance API (SimpleLoggerConsole)

Каждый вызов метода дублируется в нативную консоль браузера с красивым форматированием Aurora.

const logger = this.loggerService.getLogger(this);

logger.info('Сообщение', payload?); // Информационный лог
logger.warn('Предупреждение', details?); // Желтый статус
logger.error('Ошибка', errorContext?); // Красный статус
logger.debug('Отладка', rawData?); // Серый статус (Debug)

4. Терминальные команды

Консоль поддерживает интерактивный ввод. Список основных команд:

  • help — Список всех утилит
  • auth-status — Статус User, роли, токены
  • clear-cache — Очистка Local/Session Storage
  • toggle-theme — Переключение тем Dark/Light
  • clear — Очистка экрана консоли
  • echo [text] — Вывод текста в терминал

5. Примеры использования

Внедрение в бизнес-компонент:

import { inject, signal } from '@angular/core';
import { LoggerConsoleService } from '@shared/services/logger-console.service';

export class UserProfileComponent {
  private loggerService = inject(LoggerConsoleService);
  private logger = this.loggerService.getLogger(this); // Инстанс с префиксом класа
  
  public userStatus = signal<'active' | 'pending'>('pending');

  constructor() {
    // Авто-отслеживание состояния сигнала
    this.loggerService.trackSignal(this.userStatus, 'Status_Watcher');
  }

  updateProfile() {
    this.logger.info('Запрос на обновление профиля');
    // ... логика ...
  }
}
