Жизненный цикл безопасности

Шаг 1: Инициация входа (Login Flow)

Запрос на /api/Auth/login. Проверка в базе AuroraIdentityDb.

Шаг 2: Генерация и "Посадка" токенов

Бэкенд генерирует Access Token (JWT) и Refresh Token (Opaque). Оба упаковываются в HttpOnly Cookies. 

// Token Storage Nuances
accessToken: HttpOnly Cookie (Stateless)
refreshToken: HttpOnly Cookie + Database Table (Session Management)

Шаг 4: Silent Refresh (Магия 401)

Автоматическое продление сессии при получении 401 ошибки без прерывания работы пользователя.

Реализация на Angular 18

State Management (Signals)

public currentUser = signal<User | null>(null);
public isAdmin = computed(() => this.currentUser()?.roles.includes('Admin'));

401 Interceptor Algorithm

  1. Intercept HttpErrorResponse.
  2. Queue requests via BehaviorSubject.
  3. Execute refreshToken().
