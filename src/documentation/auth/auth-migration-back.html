Стратегия переезда Back-end: Вариант "Shared Library"

Цель: создание NuGet-пакета или DLL, которую можно подключить к любому новому сервису за 1 минуту.

1. Разделение Контекста БД (IdentityDbContext)

Создается AuroraIdentityDbContext, который содержит только системные таблицы (Users, Roles, Sessions). Это позволяет основной БД проекта оставаться чистой.

public class AuroraIdentityDbContext : IdentityDbContext<ApplicationUser> { ... }

2. "Магическая кнопка" инициализации

Вся настройка JWT и Identity прячется в метод расширения. В новом проекте достаточно одной строки:

// В Program.cs нового проекта
builder.Services.AddAuroraSecurity(builder.Configuration);

3. Контроллеры в библиотеке

AuthController упаковывается прямо в DLL. ASP.NET Core автоматически обнаружит маршруты /api/auth/* при подключении библиотеки.

4. Общие DTO и Ответы

Все объекты обмена данными (LoginDto, AuthResponse) переезжают в библиотеку. Это гарантирует, что Фронтенд всегда будет получать данные в одном и том же формате.

5. Конфигурация (appsettings.json)

Библиотека требует стандартизированную секцию настроек:

"JwtSettings": {
    "Secret": "...",
    "Issuer": "AuroraAuth",
    "ExpiryInMinutes": 60
}
Итог: При создании нового проекта (например, "Сервер имен") вы просто подключаете DLL, и у вас "из коробки" работает вся регистрация, вход и защита маршрутов.
