🎨 Авторизация: Frontend (Angular)

Фронтенд-часть отвечает за управление состоянием пользователя, хранение метаданных и автоматическую обработку истечения токенов через перехватчики запросов.

📂 Структура файлов (src/app/auth)

services/auth.service.ts Главный сервис. Управляет Signals (isAuthenticated, isAdmin) и методами login/logout.
services/auth.interceptor.ts Перехватчик. Добавляет withCredentials и обрабатывает 401 ошибки (Refresh Flow).
services/token.service.ts Низкоуровневая работа с токенами, парсинг ролей из JWT, мониторинг времени жизни.
guards/guards.ts AuthGuard, AdminGuard - защитники маршрутов.

📦 Основные модели (src/app/auth/models)

export interface UserProfileDto {
  id: string;
  email: string;
  userName: string;
  roles: string[];
  avatarUrl?: string;
}

export interface AuthResponseDto {
  user: UserProfileDto;
  accessToken?: string;  // Используется только если не установлены HttpOnly куки
  refreshToken?: string;
}

🔄 Принцип работы Refresh Token

  1. Запрос уходит на сервер. Сервер возвращает 401 Unauthorized.
  2. AuthInterceptor ловит ошибку, проверяет, не идет ли уже процесс обновления.
  3. Вызывается authService.refreshToken().
  4. Если обновление успешно — исходный запрос повторяется с новыми куками.
  5. Если ошибка — пользователь перенаправляется на /auth/login.

💡 Связь: Фронтенд не знает секретный ключ JWT. Он лишь доверяет кукам, которые устанавливает бэкенд, и реагирует на статус-коды ответа.
