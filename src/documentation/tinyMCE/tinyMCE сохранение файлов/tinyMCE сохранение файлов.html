Сохранение файлов TinyMCE на диск

В проекте Aurora реализовано два независимых механизма сохранения данных из TinyMCE на сервер: Загрузка медиа-файлов и Прямое сохранение документов.

1. Механизм сохранения документов (Справка)

Используется для редактирования файлов .html и .md напрямую в папке src/documentation/ проекта фронтенда.

Схема работы:
  1. Фронтенд отправляет контент через IconLaboratoryService.saveToDisk().
  2. Запрос идет на эндпоинт бэкенда: POST /api/Icons/save-to-disk.
  3. Бэкенд-сервис IconLaboratoryService сопоставляет относительный путь (например, src/documentation/) с абсолютным путем проекта на диске.

2. Настройка путей (Backend)

Чтобы бэкенд знал, куда именно записывать файлы фронтенда, необходимо настроить базовый путь в appsettings.json бэкенда: 

"FrontendProjectPath": "d:\\_PROGECT\\pr_aurora_admin"

Если путь в запросе начинается с src/, бэкенд автоматически подставит значение из FrontendProjectPath в начало пути.

3. Обработка форматов

В текущей реализации IconLaboratoryService:

  • Авто-расширение: Принудительное добавление .svg отключено, что позволяет сохранять .html и .md.
  • Кодировка: Файлы записываются в кодировке UTF-8 через File.WriteAllTextAsync.
  • Права доступа: Процесс бэкенда (IIS/Kestrel) должен иметь права на запись в целевую папку фронтенда.

4. Загрузка изображений

Загрузка классических изображений (через выбор файла) идет по другому маршруту:

  • Эндпоинт: POST /api/simple-image/upload.
  • Контроллер: ImageUploadTinyController.
  • Результат: Бэкенд возвращает JSON с imageUrl, который TinyMCE вставляет в src тега img.
