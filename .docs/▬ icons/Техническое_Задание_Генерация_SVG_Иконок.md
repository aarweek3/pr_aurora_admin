# Техническое Задание: Генерация SVG Иконок "На Лету"

## 1. Общее назначение

Данный документ регламентирует стандарт создания SVG-иконок силами AI-ассистента для использования в веб-интерфейсах (админ-панели, плагины TinyMCE, кнопки управления). Основная цель — получение гарантированно работающего, предсказуемого и легкого кода без артефактов отображения.

## 2. Технические требования к коду SVG

### 2.1. Контейнер и размеры

- **ViewBox**: Строго `0 0 24 24` (стандарт Material Design / современные веб-интерфейсы), если явно не указано иное.
- **Размеры атрибутов**: Явно указывать `width="24"` и `height="24"` в теге `<svg>`, если иконка предназначена для вставки как HTML-строка (например, в TinyMCE).
- **Namespace**: Атрибут `xmlns="http://www.w3.org/2000/svg"` обязателен.

### 2.2. Чистота путей (Path Cleanliness)

- **Запрет трансформаций**: Категорически избегать атрибутов `transform`, `scale`, `translate` внутри групп `<g>` или путей `<path>`, если без них физически невозможно обойтись (например, сложный поворот на 45 градусов).
  - _Плохо:_ `<g transform="scale(0.5) translate(10,10)">...</g>`
  - _Хорошо:_ Координаты точек пути (`d="..."`) должны быть пересчитаны математически под сетку 24х24.
- **Целочисленные координаты**: Стремиться использовать целые числа или не более 1 знака после запятой (например, `12.5`). Избегать дробей вида `12.34567`.
- **Единый путь**: При возможности объединять геометрические примитивы в один тег `<path>`, если они имеют одинаковый цвет.

### 2.3. Стилизация

- **Fill (Заливка)**: Использовать `fill="currentColor"`. Это позволяет управлять цветом иконки через CSS родительского элемента (color: red/blue/etc).
- **Stroke (Обводка)**: Если иконка линейная (outline), использовать `stroke="currentColor"`, `fill="none"`, `stroke-width="2"` (стандартная толщина).
- **Отсутствие инлайн-стилей**: Не использовать теги `<style>` или атрибуты `style="..."` внутри SVG.

## 3. Принципы Дизайна (Метафоры)

- **Читаемость**: Иконка должна считываться в размере 16x16px и 24x24px.
- **Минимализм**: Убирать мелкие детали, которые превратятся в "шум" при уменьшении.
- **Отступы**: Оставлять охранное поле (padding) минимум 2px от края viewBox (рисовать в области от 2 до 22 px), чтобы иконка не "прилипала" к краям кнопки.

---

## 4. Формат запроса (Промпта) к AI

Чтобы получить результат по этому стандарту, используйте следующую структуру запроса:

1.  **Объект**: Что рисуем?
2.  **Стиль**: Заливка (Filled) или Контур (Outline/Stroke)?
3.  **Контекст**: Где будет использоваться (кнопка, меню, список)?
4.  **Детали**: Особенности (метафора).

---

## 5. Примеры запросов (для копирования)

### Пример 1: Иконка "Отменить действие" (Undo)

> **Запрос:**
> "Сгенерируй SVG иконку 'Отменить действие' (стрелка назад с закруглением).
> Стиль: Outline (контурная), толщина линий 2px.
> Требования: ViewBox 0 0 24 24, без transform, координаты пересчитать в абсолютные значения. Цвет currentColor."

### Пример 2: Иконка "Список пользователей"

> **Запрос:**
> "Нужна SVG иконка для кнопки 'Список пользователей'.
> Метафора: Силуэт человека слева, справа от него три горизонтальные линии (имитация списка).
> Стиль: Filled (заливка).
> Требования: ViewBox 0 0 24 24, максимально простой path, fill='currentColor'. Без group transforms."

### Пример 3: Иконка "Слои / Наложение" (для графического редактора)

> **Запрос:**
> "Создай SVG иконку 'Слои'.
> Метафора: Два ромба или квадрата, наложенных друг на друга со смещением.
> Стиль: Outline (строгая геометрия).
> Требования: ViewBox 0 0 24 24. Использовать только прямые линии. Убедись, что иконка центрирована и имеет отступы от краев. Код должен быть чистым, без translate."
