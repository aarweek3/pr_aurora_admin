Для того чтобы Prettier корректно работал в VS Code (особенно в проектах на Angular 19), недостаточно просто установить расширение — его нужно активировать в настройках как форматер «по умолчанию».

Вот пошаговая инструкция:

1. Установка расширения
   Откройте VS Code.

Нажмите Ctrl + Shift + X (вкладка Extensions).

Введите в поиске Prettier - Code formatter.

Нажмите Install.

2. Установка форматера по умолчанию
   Это критический шаг, иначе VS Code может продолжать использовать встроенный форматер.

Нажмите Ctrl + , (запятая), чтобы открыть настройки.

В строке поиска введите Editor: Default Formatter.

В выпадающем списке выберите Prettier - Code formatter (esbenp.prettier-vscode).

3. Настройка авто-форматирования при сохранении
   Чтобы код выравнивался сам, когда вы нажимаете Ctrl + S:

В тех же настройках введите Editor: Format On Save.

Поставьте галочку напротив этого пункта.

4. Настройка для Angular (через файл .prettierrc)
   В проектах Angular важно соблюдать определенные правила (например, использовать одинарные кавычки). Создайте в корне вашего проекта файл .prettierrc (без расширения) и добавьте туда эти настройки:

JSON

{
"singleQuote": true,
"fullInference": true,
"trailingComma": "all",
"printWidth": 120,
"tabWidth": 2,
"semi": true
} 5. Решение проблем (если не работает)
Если вы нажимаете сохранить, а код не меняется:

Проверьте статус-бар: Внизу справа в окне VS Code должна быть надпись Prettier с маленькой галочкой. Если там восклицательный знак — нажмите на него, чтобы увидеть ошибку.

Конфликт с ESLint: Если у вас установлен ESLint, убедитесь, что они не "дерутся". Обычно для Angular проектов рекомендуется устанавливать плагин eslint-config-prettier.

Совет по Markdown: Как мы обсуждали ранее, ссылки в .md файлах на ваши файлы (как в этом примере) будут выглядеть гораздо лучше, если Prettier их правильно отформатирует (уберет лишние пробелы и выровняет отступы).

Если файлы .prettierrc.json и .prettierignore уже есть в корне вашего проекта Angular 19, значит, проект уже готов к работе. Чтобы VS Code начал их использовать, нужно выполнить несколько действий.

1. Настройка VS Code
   Само наличие файлов не заставляет редактор использовать их автоматически. Нужно указать это в настройках:

Откройте Settings (Ctrl + ,).

Найдите Editor: Default Formatter и выберите Prettier - Code formatter.

Найдите Editor: Format On Save и убедитесь, что галочка включена.

Убедитесь, что настройка Prettier: Require Config включена (это заставит Prettier использовать именно ваши файлы .prettierrc.json).

2. Проверка файлов
   Убедитесь, что ваши файлы содержат корректные данные для Angular проекта:

В .prettierrc.json (пример стандартных настроек):

JSON

{
"singleQuote": true,
"trailingComma": "all",
"printWidth": 120,
"tabWidth": 2
}
В .prettierignore (чтобы не форматировать лишнее):

Фрагмент кода

# Игнорировать артефакты сборки и зависимости

dist
node_modules

# Можно добавить вашу папку с копиями, если она внутри проекта

D:/COPY plagin 3. Как принудительно отформатировать всё
Если вы хотите применить настройки ко всем файлам в src прямо сейчас:

Откройте терминал в VS Code (Ctrl + ~).

Запустите команду:

Bash

npx prettier --write src/
Это перепишет все файлы в папке src в соответствии с вашим .prettierrc.json.

4. Если Prettier всё равно не работает
   Посмотрите в правый нижний угол окна VS Code (строка состояния).

Если там написано Prettier с галочкой — всё ок.

Если там два восклицательных знака или надпись не видна — кликните по ней. Откроется панель Output, где будет написана конкретная ошибка (например, лишняя запятая в .json файле).
