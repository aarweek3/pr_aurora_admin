# Глобальная Стилистическая Архитектура Проекта Aurora (Aurora Admin Design System)

Данный документ описывает современную архитектуру стилей проекта **Aurora Admin**, основанную на стандарте **v2.0 (2026 Standard)**. Система стилей построена с использованием CSS Cascade Layers (каскадных слоев), дизайн-токенов через CSS-переменные, соглашения BEM (Block Element Modifier) и глубокой интеграции с фреймворком компонентов **NG-ZORRO (Ant Design)**.

---

## 1. Архитектура каскадных слоев (CSS Cascade Layers)

Специфичность и приоритетность стилей в проекте строго контролируются с помощью каскадных слоев CSS (`@layer`), определенных в [layers.scss](file:///d:/_PROGECT/pr_aurora_admin/src/styles/abstracts/_layers.scss):

```scss
@layer base, vendor, components, utilities;
```

Это гарантирует отсутствие конфликтов специфичности селекторов и четкий приоритет:

1. **`base` (Базовый слой)**:
   * Дизайн-токены (CSS-переменные).
   * Сброс стилей (CSS Reset) и глобальные правила типографики.
2. **`vendor` (Слой сторонних библиотек)**:
   * Переопределение глобальных стилей и тем NG-ZORRO (Ant Design) и TinyMCE.
3. **`components` (Слой компонентов)**:
   * Уникальные BEM-компоненты Aurora (кнопки, поля ввода, переключатели, панели).
4. **`utilities` (Слой утилит)**:
   * Финальные служебные классы высокой специфичности (например, скрытие элементов, отступы).

## 4. Глобальные BEM Компоненты (Aurora UI)

Все базовые контроллеры стилизованы независимо от внешних библиотек в слое `components` для повторного использования:

### 4.1. Кнопки (Buttons) — [_button.scss](file:///d:/_PROGECT/pr_aurora_admin/src/styles/components/_button.scss)
* **`.av-btn`**: Базовая кнопка.
  * *Модификаторы типа*: `--primary`, `--default`, `--dashed`, `--text`, `--danger`.
  * *Модификаторы размеров*: `--small` (высота 24px), `--large` (высота 40px).
  * *Модификаторы формы*: `--shape-circle` (круглая), `--shape-round` (эллипс).
  * *Состояния*: `--loading` (блокировка, встроенный спиннер).

* **Premium Hero-Стиль**:
  * **`.av-btn-hero`**: Крупная премиальная кнопка (48px) с заливкой градиентом основного цвета, выразительной тенью и плавной анимацией приподнимания (`translateY(-2px)`) при наведении. Используется для главных целевых действий и сохранения форм.
  * **`.av-btn-secondary`**: Контурная премиальная кнопка (48px) для второстепенных действий (отмена, закрытие).

* **Специальные классы кнопок**:
  * **`.arrow-back-green`**: Универсальный класс премиальной кнопки "Назад". Представляет собой насыщенный зеленый круг (`#2eb85c`) с белой стрелкой, который становится ярче при наведении, плавно масштабируется (`scale(1.08)`) и смещает иконку влево для создания приятного микровзаимодействия.

### 4.2. Поля ввода (Inputs) — [_input.scss](file:///d:/_PROGECT/pr_aurora_admin/src/styles/components/_input.scss)
* **`.av-input`**: Универсальное поле ввода.
  * *Модификаторы размеров*: `--small`, `--large`.
  * *Состояния ошибок*: `--error` (красная рамка), `--warning` (желтая рамка), `--success` (зеленая рамка).
  * *Модификаторы визуального стиля*: `--filled` (заливка серым без обводки, белеет при фокусе), `--borderless` (без границ, только нижнее подчеркивание).
* **`.av-input-group`**: Объекты группировки инпутов с префиксами/суффиксами (иконки, валюты, единицы измерения):
  * `.av-input-group__prefix`
  * `.av-input-group__suffix`

### 4.3.iOS-Переключатели (Toggles) — [_toggle.scss](file:///d:/_PROGECT/pr_aurora_admin/src/styles/components/_toggle.scss)
Премиальные переключатели с плавной анимацией скольжения:
* **`.av-toggle`**: Базовый контейнер (скрывает оригинальный checkbox).
  * `.av-toggle__slider`: Фоновый бегунок. Меняет цвет на `--av-color-primary` при активации.
* **`.av-toggle-wrapper`**: Готовый flex-компонент с текстом подписи справа от переключателя:
  * `.av-toggle-wrapper__label`

### 4.4. Структура форм (Form Fields) — [_form-field.scss](file:///d:/_PROGECT/pr_aurora_admin/src/styles/components/_form-field.scss)
* **`.av-form-field`**: Логический блок поля ввода в форме.
  * `.av-form-field__label`: Подпись поля (поддерживает индикатор обязательности через псевдокласс `::after`).
  * `.av-form-field__container`: Контейнер инпута с поддержкой префиксов/суффиксов.
  * `.av-form-field__message`: Контейнер сообщений валидации:
    * `.av-form-field__hint`: Подсказка для заполнения.
    * `.av-form-field__error`: Текст ошибки валидации (подсвечивается красным).

## 5. Переопределение NG-ZORRO (Ant Design Overrides)

Глобальная интеграция библиотеки Ant Design и UI-токенов Aurora реализована в файле [_ant-override.scss](file:///d:/_PROGECT/pr_aurora_admin/src/styles/abstracts/_ant-override.scss) в слое `vendor`:

### 5.1. Связывание токенов
Внутренние переменные Ant Design жестко привязаны к CSS-переменным Aurora:
* `--ant-primary-color` ➔ `var(--av-color-primary)`
* `--ant-success-color` ➔ `var(--av-color-success)`
* `--ant-border-radius-base` ➔ `var(--av-radius-base)`

### 5.2. Глобальные кастомные фиксы Ant-компонентов
* **`.ant-card`**: Установлен закругленный радиус `--av-radius-lg` (`12px`) для более мягкого, премиального вида.
* **`.ant-btn`**: Скруглены по стандарту Aurora. Убрана лишняя базовая тень.
* **`.ant-input`, `.ant-select-selector`**: Приведены к единому радиусу скругления.

### 5.3. Премиальный стиль Кнопки "Назад" в Page Header
Глобально переопределена стандартная кнопка возврата `.ant-page-header-back-button` для всех страниц админ-панели:
* **Форма**: Идеально круглая кнопка размером `32x32px` с нулевыми внутренними отступами.
* **Цвет**: Насыщенный фирменный зеленый цвет (`background-color: #2eb85c !important`) с белой стрелкой (`#ffffff`).
* **Эффекты**:
  * Плавное увеличение масштаба (`transform: scale(1.08)`) при наведении.
  * Мягкое смещение иконки chevron влево (`transform: translateX(-2px)`) при hover.
  * Небольшое сжатие (`scale(0.96)`) при клике (эффект физической упругости).
  * Насыщенная зеленая неоновая тень `box-shadow` при наведении для придания премиального свечения.
* **Фикс клиппинга**: Классы `.ant-page-header-back` и `.ant-page-header-heading-left` имеют принудительный параметр `overflow: visible !important`, что полностью исключает обрезание тени круглой кнопки границами шапки.

## 7. Практическое руководство по использованию стилей

1. **Не используйте инлайн-стили (`style="..."`)**:
   Всегда применяйте токенизированные переменные из палитры (например, `color: var(--av-text-primary)`).
2. **Используйте каскадные слои в компонентах**:
   Если вы пишете уникальный стиль компонента, оборачивайте его в `@layer components { ... }` для корректного распределения приоритетов.
3. **Использование премиального стиля кнопок**:
   * Для кнопок "Сохранить", "Создать" в формах используйте класс `.av-btn-hero`.
   * Для кнопок "Отмена", "Назад" в формах используйте класс `.av-btn-secondary`.
   * Для независимых кнопок "Назад" используйте готовый класс `.arrow-back-green`.
