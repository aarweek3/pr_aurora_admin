# Техническое задание: Avalonia Desktop Parser для домашнего сервера

## 1. Общая концепция проекта

### 1.1 Цель системы

Создать desktop-приложение на Avalonia для парсинга веб-сайтов с возможностью:

- Управления сайтами и правилами извлечения данных
- Планирования и выполнения заданий парсинга
- Мониторинга процессов в real-time
- Просмотра и экспорта результатов
- Работы в фоновом режиме на домашнем сервере

### 1.2 Ключевые принципы

- **Single-user application** - один пользователь, локальный доступ
- **Offline-first** - работа без интернета (кроме парсинга)
- **Resource efficient** - оптимизация для домашнего сервера
- **User-friendly** - простой и интуитивный интерфейс
- **Extensible** - возможность добавления новых стратегий парсинга

## 2. Архитектура приложения

### 2.1 Технологический стек

- **UI Framework**: Avalonia UI 11+ (MVVM pattern)
- **Backend**: .NET 8.0
- **Database**: PostgreSQL 15+
- **ORM**: Entity Framework Core 8+
- **Parsing Engine**: AngleSharp + Playwright
- **Background Services**: .NET Hosted Services
- **Scheduling**: NCrontab для cron expressions
- **Configuration**: appsettings.json + user settings

## 3. Компоненты системы

### 3.1 UI Components (Avalonia Views)

#### 3.1.1 MainWindow

- **Назначение**: Главное окно приложения с navigation
- **Компоненты**:
  - Sidebar navigation menu
  - Content area для текущей view
  - Status bar с информацией о системе
  - System tray integration для работы в фоне

#### 3.1.2 SitesView

- **Назначение**: Управление списком сайтов для парсинга
- **Функции**:
  - CRUD операции с сайтами
  - Настройка extraction rules
  - Тестирование правил извлечения
  - Import/Export конфигураций сайтов

#### 3.1.3 JobsView

- **Назначение**: Управление заданиями парсинга
- **Функции**:
  - Создание новых заданий
  - Планирование выполнения (cron expressions)
  - Мониторинг активных заданий
  - Просмотр истории выполнения

#### 3.1.4 ResultsView

- **Назначение**: Просмотр и анализ результатов парсинга
- **Функции**:
  - Табличный просмотр данных с фильтрацией
  - Export в различные форматы (CSV, JSON, Excel)
  - Статистика по успешности парсинга
  - Просмотр raw HTML данных

#### 3.1.5 LogsView

- **Назначение**: Real-time просмотр логов системы
- **Функции**:
  - Live streaming логов
  - Фильтрация по уровню важности
  - Поиск по логам
  - Export логов в файл

#### 3.1.6 SettingsView

- **Назначение**: Конфигурация системы
- **Функции**:
  - Database connection settings
  - Parser engine настройки
  - Performance tuning
  - Backup/Restore settings

### 3.2 Core Business Logic

#### 3.2.1 Site Management

- **Entities**: Site, ExtractionRule, SelectorRule
- **Operations**:
  - Валидация URL и доступности сайта
  - Тестирование extraction rules
  - Версионирование конфигураций
  - Backup/restore настроек

#### 3.2.2 Job Management

- **Entities**: ParseJob, JobSchedule, JobExecution
- **Operations**:
  - Создание заданий из списка URL
  - Планирование повторяющихся заданий
  - Управление приоритетами выполнения
  - Отмена и рестарт заданий

#### 3.2.3 Result Management

- **Entities**: ParseResult, ExtractedData, ParseError
- **Operations**:
  - Сохранение структурированных данных
  - Агрегация статистики
  - Дедупликация результатов
  - Архивирование старых данных

### 3.3 Parsing Engine

#### 3.3.1 Strategy Pattern для типов парсинга

- **HttpParsingStrategy**: Статические HTML страницы
- **JavaScriptParsingStrategy**: SPA приложения (Playwright)
- **ApiParsingStrategy**: REST/GraphQL endpoints
- **CustomParsingStrategy**: Специфичные сайты

#### 3.3.2 Data Extraction Engine

- **CSS Selectors**: Основной механизм извлечения
- **XPath Support**: Для сложных случаев
- **Regex Patterns**: Post-processing данных
- **Custom Transformers**: Обработка специфичных данных

#### 3.3.3 Rate Limiting & Politeness

- **Per-site rate limits**: Настраиваемые ограничения
- **Robots.txt respect**: Соблюдение правил сайта
- **User-Agent rotation**: Антибан меры
- **Request delays**: Имитация человеческого поведения

### 3.4 Background Services

#### 3.4.1 JobExecutorService

- **Назначение**: Выполнение заданий парсинга в фоне
- **Функции**:
  - Queue management для заданий
  - Параллельное выполнение с ограничениями
  - Error handling и retry логика
  - Progress reporting в UI

#### 3.4.2 SchedulerService

- **Назначение**: Планировщик повторяющихся заданий
- **Функции**:
  - Cron expression parsing
  - Trigger management
  - Persistent scheduling state
  - Missed job handling

#### 3.4.3 MaintenanceService

- **Назначение**: Обслуживание системы
- **Функции**:
  - Database cleanup (старые данные)
  - Log rotation
  - Performance monitoring
  - Backup automation

## 4. Модели данных

### 4.1 Core Entities

#### Site Entity

```
Site:
  - Id: long (PK)
  - Name: string (уникальное имя)
  - BaseUrl: string (домен сайта)
  - SiteType: enum (Static, SPA, API, Custom)
  - IsActive: bool

  // Parsing configuration
  - PreferredStrategy: ParsingStrategy enum
  - RateLimitPerMinute: int (по умолчанию 30)
  - DelayBetweenRequests: TimeSpan
  - MaxConcurrentRequests: int

  // Headers & Authentication
  - CustomHeaders: Dictionary<string, string>
  - UserAgent: string
  - RequiresAuthentication: bool
  - AuthenticationConfig: JSON

  // Extraction rules
  - ExtractionRules: List<ExtractionRule>

  // Metadata
  - CreatedAt: DateTime
  - UpdatedAt: DateTime
  - LastTestedAt: DateTime
  - Notes: string
```

#### ExtractionRule Entity

```
ExtractionRule:
  - Id: long (PK)
  - SiteId: long (FK)
  - Name: string (название поля)
  - Description: string

  // Selector configuration
  - SelectorType: enum (CSS, XPath, Regex)
  - Selector: string
  - Attribute: string (text, href, src, etc.)
  - IsMultiple: bool (извлекать все или первый)
  - IsRequired: bool
  - DefaultValue: string

  // Fallback selectors
  - FallbackSelectors: List<string>

  // Post-processing
  - Transformations: List<DataTransformation>
  - ValidationRules: List<ValidationRule>

  // Conditional rules
  - ApplyWhen: string (условие применения)
  - Priority: int (порядок применения)
```

#### ParseJob Entity

```
ParseJob:
  - Id: long (PK)
  - SiteId: long (FK)
  - Name: string
  - Description: string

  // Input configuration
  - InputType: enum (ManualUrls, UrlPattern, Sitemap, File)
  - InputData: JSON (URLs, patterns, file paths)
  - MaxUrls: int (ограничение количества)
  - MaxDepth: int (для crawling)

  // Execution configuration
  - Priority: int (1-10)
  - MaxRetries: int
  - TimeoutSeconds: int
  - EnableJavaScript: bool
  - FollowRedirects: bool

  // Scheduling
  - ScheduleType: enum (Once, Recurring)
  - CronExpression: string
  - NextRunTime: DateTime?
  - LastRunTime: DateTime?

  // Status
  - Status: enum (Draft, Scheduled, Running, Paused, Completed, Failed)
  - CreatedBy: string (user identifier)
  - CreatedAt: DateTime
  - UpdatedAt: DateTime
```

#### ParseResult Entity

```
ParseResult:
  - Id: long (PK)
  - JobId: long (FK)
  - ExecutionId: Guid (группировка по запуску)

  // URL information
  - OriginalUrl: string
  - FinalUrl: string (после редиректов)
  - UrlHash: string (для дедупликации)

  // Parsing result
  - Status: enum (Success, Failed, Timeout, Blocked)
  - ExtractedData: JSON (структурированные данные)
  - RawHtml: string? (опционально)
  - ParsingStrategy: enum (Http, JavaScript, etc.)

  // HTTP metadata
  - StatusCode: int
  - ResponseHeaders: JSON
  - ContentType: string
  - ContentLength: long
  - LoadTime: TimeSpan

  // Error information
  - ErrorType: string?
  - ErrorMessage: string?
  - RetryCount: int

  // Timestamps
  - CreatedAt: DateTime
```

## 5. User Experience Flow

### 5.1 Типичный workflow пользователя

#### Начальная настройка:

1. Запуск приложения → настройка database connection
2. Добавление первого сайта → тестирование extraction rules
3. Создание первого задания → запуск парсинга
4. Просмотр результатов → настройка экспорта

#### Ежедневное использование:

1. Проверка статуса активных заданий
2. Просмотр новых результатов
3. Корректировка rules при изменении сайтов
4. Экспорт данных для анализа

#### Планирование и мониторинг:

1. Настройка recurring jobs с cron expressions
2. Мониторинг performance и error rates
3. Периодическая очистка старых данных
4. Backup конфигураций и результатов

### 5.2 Error Handling Strategy

#### User-facing errors:

- **Network issues**: Автоматический retry с экспоненциальным backoff
- **Site changes**: Уведомление об изменении структуры + suggestions для fixes
- **Resource limits**: Предупреждения о нехватке памяти/диска
- **Configuration errors**: Валидация в real-time с подсказками

#### Background errors:

- **Silent failures**: Логирование без interruption пользователя
- **Critical failures**: Уведомления в UI + возможность manual intervention
- **Rate limiting**: Автоматическая адаптация delays
- **Data corruption**: Automatic rollback + manual recovery options

## 6. Performance & Scalability Considerations

### 6.1 Для домашнего сервера

- **Memory optimization**: Streaming processing для больших datasets
- **CPU efficiency**: Асинхронная обработка с semaphore limits
- **Disk space**: Автоматическая архивация старых результатов
- **Network politeness**: Respect for target sites с configurable delays

### 6.2 Database optimization

- **Indexing strategy**: Composite indexes для частых queries
- **Partitioning**: Time-based partitioning для results table
- **Connection pooling**: Efficient EF Core configuration
- **Query optimization**: LINQ best practices + custom SQL где нужно

### 6.3 UI responsiveness

- **Background processing**: Все длительные операции в background
- **Progress reporting**: Real-time updates без blocking UI
- **Lazy loading**: Виртуализация для больших lists
- **Caching**: In-memory кеширование для frequently accessed data

## 7. Security & Privacy

### 7.1 Local security

- **Database encryption**: Опциональное шифрование sensitive data
- **Configuration protection**: Encrypted storage для credentials
- **Log sanitization**: Удаление sensitive data из логов
- **Backup security**: Encrypted backups с password protection

### 7.2 Web scraping ethics

- **Robots.txt compliance**: Автоматическая проверка и соблюдение
- **Rate limiting**: Conservative defaults с возможностью настройки
- **User-Agent honesty**: Правдивые User-Agent strings
- **GDPR awareness**: Инструменты для data minimization

## 8. Deployment & Maintenance

### 8.1 Installation process

1. **Prerequisites check**: .NET Runtime, PostgreSQL availability

4. **Service registration**: Опциональная установка как Windows Service

### 8.2 Updates & Migrations

- **Auto-update mechanism**: Проверка новых версий + download
- **Database migrations**: EF Core migrations с rollback support
- **Configuration migrations**: Automatic upgrade старых configs
- **Backup before update**: Automatic backup перед major updates

### 8.3 Monitoring & Diagnostics

- **Health checks**: Database connectivity, disk space, memory usage
- **Performance counters**: Requests/second, error rates, response times
- **Alerting**: Configurable alerts для critical issues
- **Diagnostic exports**: Support bundle generation для troubleshooting
