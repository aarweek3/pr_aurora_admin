# Инструкция по встраиванию Aurora Image Studio в любую Angular-форму

Для интеграции мы используем сервис `NzModalService` (из Ng-Zorro) для открытия компонента `AvImageStudioModalComponent`.

---

### 1. Подготовка (Импорты)

В вашем компоненте, где находится форма, добавьте необходимые импорты:

```typescript
import { NzModalService } from "ng-zorro-antd/modal";
import { AvImageStudioModalComponent } from "@shared/components/av-image-studio-modal/av-image-studio-modal.component";
import { AvImageUploadResult } from "@shared/components/av-image-studio-modal/models/av-image-studio-modal.model";
```

### 2. Инъекция сервиса

Добавьте сервис модальных окон в конструктор или через `inject`:

```typescript
export class YourComponent {
  private modal = inject(NzModalService);

  // Переменная для хранения ссылки на изображение в вашей форме
  imageUrl: string = "";
}
```

### 3. Метод для открытия Студии

Создайте функцию, которая будет вызывать модальное окно. Она принимает текущее изображение и настройки.

```typescript
openImageStudio() {
  const modalRef = this.modal.create({
    nzTitle: 'Aurora Image Studio',
    nzContent: AvImageStudioModalComponent,
    nzData: {
      imageUrl: this.imageUrl, // Текущее фото (если есть)
      aspectRatio: 1.77,       // Опционально: зафиксировать пропорции (например, 16:9)
      metadata: {
        altText: 'Описание фото',
        titleText: 'Заголовок'
      }
    },
    nzWidth: 1200,             // Рекомендуемая ширина
    nzFooter: null,            // Важно: футер управляется внутри Студии
    nzDraggable: true,         // Разрешить перетаскивание
    nzMaskClosable: false,     // Запретить закрытие кликом по фону (защита от потери правок)
    nzStyle: { top: '30px' }
  });

  // Обработка результата после нажатия кнопки "Готово"
  modalRef.afterClose.subscribe((result: AvImageUploadResult | undefined) => {
    if (result && result.dataUrl) {
      // result.dataUrl — это относительный путь к файлу на сервере (напр. /uploads/images/abc.jpg)
      this.imageUrl = result.dataUrl;

      // Если у вас Reactive Forms:
      // this.form.get('image')?.setValue(result.dataUrl);

      console.log('Данные SEO:', result.metadata);
    }
  });
}
```

### 4. Вызов из HTML

Добавьте кнопку или клик по превью изображения:

```html
<!-- Вариант с кнопкой -->
<button nz-button (click)="openImageStudio()"><span nz-icon nzType="picture"></span> Выбрать / Редактировать фото</button>

<!-- Вариант с кликом по самому изображению -->
<div class="image-preview" (click)="openImageStudio()">
  <img [src]="imageUrl || 'assets/placeholder.png'" />
</div>
```

---

### Параметры `nzData`, которые принимает Студия:

| Параметр      | Тип      | Описание                                                                                                |
| :------------ | :------- | :------------------------------------------------------------------------------------------------------ |
| `imageUrl`    | `string` | Прямая ссылка или Base64 изображения для редактирования.                                                |
| `aspectRatio` | `number` | Число (например, `1`, `1.5`, `1.77`). Если передано, рамка кропа будет заблокирована в этих пропорциях. |
| `metadata`    | `object` | Объект с полями `altText`, `titleText`, `caption` для предзаполнения полей SEO.                         |

---

### Что возвращает Студия (`AvImageUploadResult`):

После успешного сохранения вы получите объект:

- `dataUrl`: Ссылка на загруженный файл (уже сохранен на сервере).
- `file`: Объект `File` (если нужно отправить его куда-то еще вручную).
- `width`/`height`: Итоговые размеры.
- `metadata`: Все заполненные SEO-поля (alt, title, описание, настройки ссылки).

**Совет:** Для удобства в проекте уже есть готовый компонент `<av-image-picker>`, который использует эту логику внутри себя. Его можно просто вставить как `[(ngModel)]="myImageUrl"`.
